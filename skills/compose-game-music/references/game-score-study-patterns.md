# Game Score Study Patterns

Use this after studying actual game music scores, official excerpts, legal sheet books owned by the user, public analyses, or short analytical excerpts. Do not copy melodies, basslines, voicings, or arrangements. Extract reusable mechanisms and convert them into original design rules.

## Core Lesson

Do not solve music by one technique: not "just sustain", not "just more notes", not "just a scale", not "just instruments", and not "just a chord progression". Real game cues work through a combination:

- Motive identity: a small shape the listener can recognize.
- Rhythmic grammar: a pulse/accent pattern that matches player motion.
- Bass behavior: pedal, walk, ostinato, answer, or pressure.
- Harmony behavior: when chords change, what stays, what pulls, what resolves.
- Voice interaction: melody, counterline, inner motion, and support do different jobs.
- Texture behavior: point, line, and plane are balanced, not stacked randomly.
- Register and timbre: each part occupies a useful range and distance.
- Gameplay function: the loop supports action, place, fatigue tolerance, and SFX.

Before composing, write the combination as a one-line formula:

`scene = motive + pulse + bass function + harmonic behavior + voice interaction + texture + loop return`

If the formula has only one strong term, the cue will sound like a trick, not a composition.

## Study Cards From Game Music Sources

These cards are abstracted from publicly discussed game music examples and official/critical summaries. They are not prescriptions and do not contain reusable copyrighted melodies.

### Action Platformer Overworld

- Sources to study: Super Mario Bros. Ground Theme, later Mario overworld arrangements, officially licensed sheet folios when available.
- Mechanism: catchy short cells, syncopation, dance-derived rhythm, call/answer contour, rests, and immediate fit to jumping/running timing.
- Combination: melody is rhythmic, bass is buoyant, harmony is simple, accompaniment leaves SFX space, and tempo can react to timer pressure.
- Transfer rule: for movement-heavy gameplay, make the motif physically playable: the accent pattern should feel like the character's body, not like background decoration.

### Adventure Field / Area Identity

- Sources to study: Zelda overworld and area themes, Ocarina of Time area music, ocarina-song transformations.
- Mechanism: place identity comes from contrast between areas, limited-note motifs, variation of learned songs, and music that approaches as environment identity rather than only character theme.
- Combination: small playable motif + environmental harmony + register/color contrast + loop that avoids fatigue.
- Transfer rule: define what tells the player "where I am" within 5 seconds, then vary it so the field does not exhaust the listener.

### RPG Character / Story Leitmotif

- Sources to study: Final Fantasy character themes, victory fanfare, Prelude evolution, recurring themes across scenes.
- Mechanism: memorable motif is reused through changes in tempo, harmony, instrumentation, register, and dramatic context.
- Combination: theme identity remains stable while accompaniment, mode, orchestration, and cadence change to match scene meaning.
- Transfer rule: write a motif that survives rearrangement; if it only works in one timbre, it is not strong enough.

### Ambient Platformer / Environmental Depth

- Sources to study: Donkey Kong Country water/forest cues and David Wise interviews.
- Mechanism: atmosphere is not only noise or pad; it is an engineered palette where waveform choice, slow harmonic motion, melodic fragments, and environmental pacing support level identity.
- Combination: timbre experiment + slow contour + sparse percussion or pulse + clear environment image + long fatigue-safe loop.
- Transfer rule: ambience still needs a musical object in the middle distance: a contour, bass gravity, or recurring interval that gives memory to the space.

### Orchestral Adventure Scale

- Sources to study: Super Mario Galaxy orchestral cues and interviews/articles on its change from "cute" to "space-like/cool".
- Mechanism: genre palette changed because the scene scale changed; orchestration was chosen to express space and grandeur while preserving Mario's active rhythm.
- Combination: large ensemble color + clear upbeat rhythmic identity + heroic/fantasy register spread + gameplay-compatible timing.
- Transfer rule: orchestration must serve scale and motion together; large sound without motion becomes wallpaper.

## Combination Analysis Grid

For every studied score or cue, fill this grid before composing:

| Dimension | What to extract | Failure if ignored |
| --- | --- | --- |
| Motive | contour, rhythm fingerprint, repeat/answer pattern | decorative notes |
| Pulse | subdivision, accent group, swing/straight feel | scene-neutral rhythm |
| Bass | role and motion timing | root-note blocks |
| Harmony | harmonic rhythm, cadences, pedals, modal color | scale-label music |
| Inner voice | common tones, suspensions, contrary motion, fills | disconnected blocks |
| Counterline | answer, imitation, antagonist, echo | random second melody |
| Texture | point/line/plane balance and density curve | dots or fog |
| Timbre | instrument role, register, articulation, distance | palette cosplay |
| Loop | return gesture and fatigue control | obvious restart |
| Gameplay | movement, SFX space, state changes | non-interactive track |

## Composition Procedure From Study

1. Pick 2-3 reference mechanisms, not whole pieces.
2. Convert each mechanism into a rule, not notes.
3. Compose a new motif with different contour, pitch content, rhythm, and phrase placement.
4. Build a two-voice skeleton: melody plus bass or melody plus harmonic rhythm.
5. Add inner motion: common tone, suspension, passing tone, contrary motion, or rhythmic fill.
6. Add counterline only if it has a scene job.
7. Add plane only after the skeleton works.
8. Add timbre and effects last.
9. Test the result on plain piano/square waves and with SFX imagined over it.

## Evidence From Theory And Research

Use these as practical composition tests:

- Repetition must form hierarchy: motif, phrase, and section should interact rather than repeat at one flat level.
- Loops need recognizable patterning and enough variation to avoid fatigue.
- Voice leading matters even in game loops: hold common tones, move inner voices by step where possible, and avoid every layer attacking/releasing together unless that is the deliberate effect.
- Nonchord tones need jobs such as passing, neighbor, suspension, anticipation, pedal, appoggiatura, or color; unresolved accidents usually read as random wrongness.
- Adaptive music should specify whether change is vertical layering, horizontal re-sequencing, algorithmic generation, or SFX/music synchronization.

## Anti-Single-Trick Gate

Before rendering, answer:

- What is the motif doing?
- What is the rhythm doing?
- What is the bass doing?
- What is the harmony doing?
- What is the inner voice doing?
- What is the texture doing?
- What is the loop return doing?
- What is the gameplay function?

If three or more answers are "nothing", "just sustaining", "just filling", "just atmosphere", or "just following chords", rebuild.

## Copyright Boundary

Allowed:

- Mentioning studied works and broad mechanisms.
- Using official/public summaries, interviews, and short analytical excerpts.
- Using user-owned/licensed sheet books for private analysis when the user provides them.
- Creating original motifs from abstracted mechanisms.

Not allowed:

- Copying or storing full copyrighted game scores.
- Reproducing recognizable melodies, basslines, chord voicings, or arrangements.
- Building a local training library from unauthorized sheets or MIDIs.
- Claiming a new cue is "in the style of" a specific living composer or exact soundtrack.
